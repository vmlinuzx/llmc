You are an implementation agent working on the LLMC RAG core.

Goal: Implement the first step of the LLMC RAG result envelope for navigation-style tools (search, where-used, lineage) without changing existing CLI behavior.

Context:
- LLMC uses a RAG stack with:
  - Indexer, enrichment pipeline, embedding pipeline.
  - RAG Nav graph + status metadata.
  - A context gateway that will decide when to use RAG vs. local fallback.
- We need a shared envelope type for RAG tool responses so:
  - MCP servers can expose consistent metadata.
  - Desktop Commander can reason about freshness and fallbacks.
  - Future clients can safely handle errors and degraded modes.

Constraints:
- Step 1 MUST NOT change behavior of existing CLIs (`tools.rag.cli`, `tools.rag_nav.cli`).
- No new network calls, daemons, or side effects – this is a pure type/envelope addition.
- Keep the module small, focused, and internal for now.

What to implement:
- A new module `tools/rag/nav_meta.py` that defines:
  - `RagToolStatus` = Literal["OK", "FALLBACK", "ERROR"]
  - `RagToolSource` = Literal["RAG_GRAPH", "LOCAL_FALLBACK", "NONE"]
  - `RagToolMeta` dataclass:
      - status: RagToolStatus (default "OK")
      - error_code: Optional[str]
      - message: Optional[str]
      - source: RagToolSource (default "RAG_GRAPH")
      - freshness_state: FreshnessState (default "UNKNOWN")
      - index_status: Optional[IndexStatus] (snapshot of status used to route)
      - method: `to_dict()` that flattens the dataclass, delegating to `IndexStatus.to_dict()` if present.
  - `RagResult[T]` generic dataclass:
      - meta: RagToolMeta
      - items: Sequence[T] (default empty tuple)
      - method: `to_dict()` that converts items via `to_dict()` or `_asdict()` if available, otherwise returns the item as-is.
  - Helper constructors:
      - `ok_result(items, source="RAG_GRAPH", freshness_state="FRESH", index_status=None, message=None)`
      - `fallback_result(items, freshness_state="STALE", index_status=None, message=None)`
      - `error_result(error_code, message, freshness_state="UNKNOWN", index_status=None)` → returns `RagResult[dict]` with empty items and error meta.

Integration notes:
- Import `FreshnessState` and `IndexStatus` from `tools.rag.freshness`.
- No changes to `tools.rag.cli` or `tools.rag_nav` in this step.
- Optional: add a small `tools/rag/tests/test_nav_meta.py` with sanity checks.

Deliverable:
- The new `tools/rag/nav_meta.py` module implemented as above, passing `python -m tools.rag.nav_meta` import checks and (optionally) basic tests.
